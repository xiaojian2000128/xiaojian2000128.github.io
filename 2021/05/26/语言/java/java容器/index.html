java容器方面的知识, 包括数组, Collection, Map

数组

数组的特点

  1. 一旦初始化了,容量不可以改变
  2. 一旦定义好后,数组里面元素的类型,就确定了(支持多态)

数组的弊端

  1. 容量固定
  2. 只能获取数组的容量,不能直接获取数组中元素的个数
  3. 对于插入和删除的效率不高

数组转Collection

Arrays.asList:

  1. 如果参数是基本类型的数组,只会被看成一个数组,
  2. 如果是引用数组则可以看成多个元素,通常用于,已经声明的数组转Collection
  3. 这个函数的参数是不定长参数,可以直接写多个引用对象,通常用于创建一个Collection,用于去填充另外一个Collection
  4. 这个方法创建的Collecion不能进行能修改其长度的操作,因为这个list本质还是一个数组

Collection的方法

增加: add, addAll

删除: remove, removeAll, retainAll, clear

包含: contains, containsAll

大小: size

是否为空: isEmpty

转数组: toArray

  1. 不传参数,默认是Object数组
  2. 传对象数组,则转为对象数组
  3. 转数组的方法无法转成基本类型数组

迭代: iterator

  1. 实现了iterable接口,必须要实现iterator方法
  2. 实现iterable接口,是增强for/in的充分不必要条件,数组 没有实现iterable接口仍可以使用增减for/in

重写Object: equals, toString, hashCode

  1. equals:List要求元素顺序相同, Set不要求元素顺序相同

迭代器

为容器提供一种统一的遍历方式,同时又不暴露容器内部实现细节

拥有的方法

hasNext, next, remove

  1. remove已经被默认实现了,不用再被实现

  2. IllegalException:

    1. 如果不掉用next方法就使用remove
    2. 如果已经调用了remove,接着再调用remove

List接口

特点: 有序,可重复

实现类: ArrayList,LinkedList,Vector

三者的异同

三者的相同: 存储特点都是有序,可重复,都实现了List接口

ArrayList和Vector的异同:

  • 相同: 底层都是用数组实现的,
  • 不同: ArrayList是从jdk1.2开始才有的,Vector从1.0就有了,ArrayList是线程不安全的,效率高,Vector是线程安全的,效率低,在jdk1.7版本中,使用时,都会直接创建一个10大小的数组,但在1.8中,ArrayList数组的创建被推迟到放入第一个元素时.在扩容时,ArrayList的大小变为原来的1.5倍,如果容量扩容后还是不够,则容量直接变为能容下元素的大小,Vector的大小在不使用增长量的时候,大小变为原来的2倍,如果容量扩容后还是不够,则容量同样是直接变为能容下元素的大小.为了避免他们两者的扩容带来的性能损耗,可以在构造时指定大小
  • tip: StringBuffer中的数组是变为原来的2倍+2

ArrayList和LinkedList的异同:

  • 相同: 都是从jdk1.2才开始出现
  • 不同: ArrayList底层使用数组实现的,对于查找效率,比较高,但对于删除和插入效率比较低,每次删除和插入元素,都会导致数组内其它元素的前移和后移,导致性能消耗,LinkedList底层使用的是双向链表,拥有一个静态内部类Node,和两个Node的成员变量作为头指针和尾指针,对于查找效率较低,但对于插入和删除效率较高,只需要修改前后节点的指针

List接口的方法

因为List接口是有序的,所以List在Collection接口基础上增加了一些关于索引的方法

增加: add(int index, Object ele), addAll(int index, Collection coll)

删除: remove(int index)

  1. 这个方法必须传基本数据类型

修改: set(int index, Object o)

查找: get(int index), IndexOf(Object o), lastIndexOf(Object o)

切分: subList(int fromIndex, int toIndex)

  1. 左边右开, 返回一个新的List, 不会对原本的List进行修改

Set接口

特点: 无序, 不可重复

实现类: HashSet, LinkedHashSet, TreeSet

三种实现类的特点

HashSet: 可以存储null值,底层用的是HashMap,放入元素时,只放入键, 值是一个静态的Object变量.在1.7中,底层使用的是数组加链表, 当发生冲突时, 新添加的元素会在原本元素的前边,使用时,会直接创建一个16大小的数组,在1.8中,底层使用的是数组加链表加红黑树,在使用链表时,新添加的元素在原本元素的后面, 当链表的长度大于8且数组的大小64时, 链表会自动转为红黑树, 如果不大于64, 则只会进行扩容操作,使用时数组在放入第一个元素的时候创建.使用HashSet需要重写hashCode和equals方法.

LinkedHashSet: 是HashSet的子类, 可以按插入顺序,遍历集合内的元素,因为有指向前一个和后一个元素的指针

TreeSet: 可以按照插入元素指定的属性进行排序,底层用的是TreeMap,用的是红黑树,插入的元素必须要实现自然排序Comparable或定制排序Comparator,在TreeSet中判断两个元素是否相等,只与Comparable或Comparator的比较结果有关,不与equals和hashCode有关

HashSet添加对象的过程

先通过对象的hashCode方法获得对象的hash值,通过hash值算出对象在数组中位置

  1. 如果该位置没有对象,就直接放入
  2. 如果有对象,这比较这两个元素的hashCode是否相等
    1. 如果不相等,则进行添加
    2. 如果相等,在调用equals方法进行比较
      1. 如果不相等,则进行添加
      2. 如果相等,则不进行添加

在1.7中, 添加的元素,会占据数组中的位置,用一个引用指向原来的元素.在1.8中,添加的元素,会链接在原本元素的后面.同时因为判断两个元素是否相等要用到hashCode和equals两个方法,要保证这两个方法的一致性,在equals中用的方法在hashCode中也要尽量使用

Map接口

特点: 映射, 存储的是键值堆

实现类: HashMap, LinkedHashMap, TreeMap, Hashtable, Properties

HashMap和Hashtable的区别

HashMap是线程不安全的,效率高,可以存储null-null的键值对,Hashtable是线程安全的,效率低,不能存储为null的键和值

五种实现类

HashMap: 线程不安全,可以存储null的键和值, 在1.7中, 底层用的是数组加链表, 使用时会直接创建一个16大小的Entry数组.在1.8中, 底层使用的是数组加链表加红黑树. 使用时Node数组在第一次放入键值对的时候创建, 链表的长度大于8且数组的长度大于64时, 链表会转为红黑树.如果链表的长度只是大于8但是数组的长度小于64, 只会对数组进行扩容, 扩容为原来的2被

LinkedHashMap: 是HashMap的子类, 但是节点对象,多了一个指向前一个对象的成员变量before和一个指向后一个对象的成员变量after, 可以按照添加的顺序进行遍历, 遍历的效率也比HashMap高

TreeMap: 可以按元素指定的属性进行排序, 底层用的红黑树, 考虑对象是否相等时, 使用的是定制排序Comparator或自然排序Comparable的方法, 不会用到元素的equals方法

Hashtable: 线程安全, 效率低, 不能存放null的键和null的值

Properties: 是Hashtable的子类, 键值都是String对象, 主要用于处理配置文件

tip: 对于HashMap和Hashtable来说, 元素要重写hashCode和equals方法, 而且尽量保持他们一致;对于TreeMap, 元素要实现自然排序或定制排序. Map中键的集合用的Set, 值的集合用的Collection, 键值对的集合用的是Set

Map接口的方法:

增加/修改: put, putAll

查找: get, getOrDefault

删除: remove, clear

是否为空: isEmpty

大小: size

包含: containsKey, containsValue

相等: equals

  • 对于HashMap顺序可以不想等, 对于TreeMap顺序必须相等
